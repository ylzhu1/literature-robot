Quickstart

配置你自己的文献机器人

这个页面面向第一次使用的人。正常使用 Windows 发布版时，不需要安装 Python，也不需要打开命令行。 你只需要下载 ZIP、打开这个指南、再打开配置软件。

推荐使用顺序 先打开本页面，再打开 LiteratureAgentSetup.exe。在软件左侧按顺序配置： Model API、Topics & Filter、Notifications、Run a Test、Schedule。

1. 下载与打开

1

下载正确的 ZIP

请在 GitHub 的 Releases 页面下载 LiteratureAgent-Windows.zip。不要用绿色 Code 按钮下载源码 ZIP，那个不是给普通用户直接运行的。

2

解压到普通文件夹

推荐解压到类似 Documents\LiteratureAgent 的本地文件夹。不要直接在压缩包里运行，也不要放在临时下载目录里长期使用。

3

打开配置软件

双击 LiteratureAgentSetup.exe。如果 Windows 提示“未知发布者”，这是因为 EXE 没有代码签名；确认文件来自你的官方 GitHub Release 后再继续。

2. Model API 应该怎么填

模型 API 用来把英文摘要写成中文摘要和深度解读。只要服务商兼容 OpenAI Chat Completions 格式，一般都可以使用。

软件字段 应该填写什么 例子
API Key 模型服务商给你的密钥。它是私密信息，不要发到 GitHub，也不要截图公开。 sk-...
Base URL API 地址根路径，通常应该以 /v1 结尾。不要填完整的 /chat/completions 路径。 https://api.openai.com/v1
Model 服务商支持的模型名称，需要和服务商文档里的名字一致。 gpt-4o-mini
这一步怎么判断成功？ 点击 Test LLM Connection。如果失败，优先检查 API Key、Base URL、模型名、网络连接，以及模型账号是否还有额度。

3. Topics & Filter 应该怎么填

这是最重要的一页。软件会先抓一批候选论文，再用关键词组和分数规则筛出真正值得发给你的论文。

打分规则

规则 分数影响
命中一个关键词组 加 2 + min(该组命中的关键词数量, 4) 分。一个组内命中很多词也不会无限加分。
命中 strong keyword 每命中一个强关键词，加 4 分。
命中 excluded keyword 每命中一个排除关键词，扣 5 分。
同时命中多个组 加交叉奖励：cross_group_bonus * (命中组数 - 1)。默认 cross_group_bonus = 3。

关键词应该怎么写

把关键词组当成“概念” 不要把所有词都堆在一个组里。更推荐分成方法、材料体系、反应过程、表征手段等不同概念。
required group 要谨慎 如果某个概念是好论文必须具备的，就把这一组勾成 required。勾了多个 required 组时，论文必须同时命中这些组。
strong keywords 放核心偏好 例如你特别关心 density functional theory、oxygen adsorption，就放到 strong keywords。
excluded keywords 放常见误报 如果总是推来废水、生物、抗菌、加工粗糙度等无关论文，就把这些词放进 excluded keywords。

默认筛选强度

{
  "min_score": 12,
  "group_min_matches": 3,
  "require_any_groups": ["oxidation"]
}

这表示：论文总分至少 12 分，至少命中 3 个关键词组，并且必须命中所有 required 组。 如果报告经常为空，可以降低 min_score 或 group_min_matches；如果无关论文太多，就提高它们，或增加 excluded keywords。

示例：硬碳 / 软碳电池负极

如果研究硬碳、软碳作为电池负极材料，不建议只写 hard carbon mechanical properties 这一类长句。 更稳的做法是把主题拆成几个概念组：

关键词组 建议内容 是否 required
carbon_material hard carbon, soft carbon, disordered carbon, non-graphitizable carbon 是
battery_context sodium-ion battery, lithium-ion battery, carbon anode, negative electrode, sodium storage 是
mechanical_property mechanical properties, Young's modulus, nanoindentation, fracture, stress, strain, cracking 通常先不勾

这样可以先保证论文确实是“碳负极”方向，再让机械性能词汇提高排序。 如果你只想要严格机械性能论文，再把 mechanical_property 也勾成 required。 常见误报可以放进 excluded keywords，例如 diamond-like carbon, DLC, steel, tribology, friction, wear, coating。

保存后搜索词也会更新 点击 Save Configuration 后，软件会根据关键词组和 strong keywords 自动更新 arXiv、Crossref、OpenAlex 的搜索词。 RSS 是固定期刊订阅源，不是关键词搜索，所以 RSS feeds 不会自动改变。

4. 飞书或邮箱通知怎么填

飞书机器人

如果你想把日报推送到飞书群，就打开 Feishu，并把群自定义机器人的 webhook 粘贴到 Webhook URL。

Webhook 相当于一个私密入口，不要公开。

邮箱

邮箱是可选项。如果只用飞书，可以关闭邮箱。若打开邮箱，建议把下面这些字段都填完整。

字段 怎么填
Email Address 发件邮箱，例如你的 QQ 邮箱。
SMTP Authorization Code 邮箱后台生成的 SMTP 授权码，不是登录密码。
Recipient 收件邮箱。最稳妥的做法是填和 Email Address 相同的邮箱地址，例如 your_email@example.com。不要留空，避免某些 SMTP 服务报“所属域名不存在”等错误。

很多邮箱的 SMTP 授权码不是登录密码，请到邮箱设置里单独生成。

这一步怎么判断成功？ 点击 Send Test Message 或 Send Test Email。只有确认收到测试消息后，再继续下一步。

5. 跑一次真实测试

打开 Run a Test，点击 Run Literature Test。它会抓论文、筛选、总结，并把一份真实日报发到你开启的通知渠道。

  • 这个过程可能需要几分钟。
  • 进度条在动就说明后台正在运行。
  • 测试模式可能会显示以前见过的论文，这是为了方便你确认格式和推送是否正常。
  • 真正的每日定时任务默认会去重，不会反复发送已经推送过的论文。

6. 设置每日自动运行

打开 Schedule，填写每天运行的时间，例如 09:00 或 18:30，然后点击 Install / Update Windows Task。

重复设置不会堆任务 如果你先设成 9 点，后来又改成 10 点，软件会删除旧任务并保留最后一次设置。

本地定时任务要求电脑在对应时间开机。如果你希望电脑关机也能运行，需要使用 README 里介绍的 GitHub Actions 云端定时方案。

修改设置后会发生什么

你修改了什么 软件会怎么处理
修改关键词组或筛选强度 点击 Save Configuration 后，新设置会覆盖旧设置。你删除的关键词不会继续生效。
修改飞书或邮箱 保存后建议重新发送测试消息，确认通知渠道仍然可用。
修改定时时间 点击 Install / Update Windows Task 后，旧时间会被替换。
之前已经推送过的论文 会继续记录在 data/literature_seen.sqlite 中。正常定时运行不会再重复发送。

常见问题

现象 可能原因 建议处理
日报没有论文 筛选太严格，或当前时间窗口没有新的高相关论文。 降低 min_score 或 group_min_matches，增加更宽泛的关键词，或等待自动回溯到更早时间窗口。
无关论文太多 关键词太宽，或者 excluded keywords 不够。 把常见误报方向加入 excluded keywords，并适当提高 min_score。
中文摘要失败，出现兜底文本 模型 API 调用失败，或者模型配置不完整。 重新点击 Test LLM Connection，检查 API Key、Base URL、模型名和账号额度。
飞书测试成功，但日报说没有新文献 推送渠道是好的，只是当前筛选没有找到合适论文。 调整关键词或筛选强度，也可以让自动回溯机制继续往前找。

隐私文件与分享

不要分享已经配置或运行过的文件夹 软件运行后会生成包含私密信息或个人历史的文件。分享给别人时，只分享官方构建出来的 LiteratureAgent-Windows.zip。
文件或文件夹 作用 能不能分享
.env 保存 API Key、飞书 webhook、邮箱授权码。 不能
data/ 保存已推送论文记录和回溯进度。 不能
reports/ 保存本地生成过的日报。 默认不要分享，除非你明确想分享某份日报。
